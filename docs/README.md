# Documentation

Living documentation for the project — kept in sync with every decision, not written once and forgotten.

- `vision/` — why this project exists, north star (frontmatter: status/date/owner)
- `concept/` — product concept, elevator pitch (frontmatter)
- `product/` — target users, MVP scope, and one doc per feature under `features/` (frontmatter)
- `architecture/` — current-state architecture overview and technical constraints
- `decisions/` — Architecture Decision Records (ADRs), sequential and append-only (frontmatter)
- `assumptions.md` — running log of assumptions made, to be revisited as they're confirmed or invalidated
- `risks.md` — risk register (frontmatter)
- `roadmap.md` — milestones and sequencing
- `glossary.md` — shared vocabulary, feeds Graphify

See [Definition of Ready](../.claude/docs-standards.md) for what must exist before implementation begins.
