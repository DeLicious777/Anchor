# Planning

Bridges finished design/architecture work to executable GitHub issues.

- `epics/` — one file per epic, each linking to the feature docs and ADRs it depends on
- `milestones.md` — sequencing and target dates for epics
- `issue-conventions.md` — labeling, naming, and template conventions for GitHub issues

Planning artifacts here should only be created once the relevant feature/architecture work has cleared the Design and Architecture workflows — not before.
